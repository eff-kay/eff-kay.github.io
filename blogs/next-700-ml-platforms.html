Abstract

We present a theoretical framework for understanding and comparing machine learning model serving systems in cloud environments. Our analysis focuses on three major platforms - Amazon SageMaker, Google Cloud Vertex AI, and Azure ML - demonstrating how their serving mechanisms can be understood through a common lens.

3. A Calculus for ML Model Serving

Core Concepts

ModelArtifact ::= (code, weights, metadata)
Container ::= (ModelArtifact, runtime, deps)
Endpoint ::= (Container, scaling_config, routing)
Version ::= (Endpoint, traffic_weight)

Operations

package : ModelArtifact → Container
deploy : Container → Endpoint
scale : Endpoint × Config → Endpoint
route : Version × Version × Weight → Version

4. Platform Analysis

4.1 Amazon SageMaker

Basic Model Deployment

Theoretical Representation:
ModelArtifact = (
    code = "sklearn_model.pkl",
    weights = "model.joblib",
    metadata = {"framework": "sklearn", "version": "1.0"}
)
Implementation:
from sagemaker.sklearn import SKLearnModel

model = SKLearnModel(
    model_data="s3://bucket/sklearn_model.tar.gz",
    role=role,
    framework_version="0.24-1",
    py_version="py3",
    entry_point="inference.py"
)

4.2 Google Cloud Vertex AI

Basic Model Deployment

Theoretical Representation:
ModelArtifact = (
    code = "model.pkl",
    framework = "tensorflow",
    metadata = {
        "framework_version": "2.8.0",
        "python_version": "3.9"
    }
)
Implementation:
from google.cloud import aiplatform

model = aiplatform.Model.upload(
    display_name='tensorflow-model',
    artifact_uri='gs://bucket/model/',
    serving_container_image_uri='us-docker.pkg.dev/vertex-ai/prediction/tf2-cpu.2-8'
)

5. Common Patterns and Differences

Key Patterns

  • Container-based deployment is universal across platforms
  • Auto-scaling based on metrics is a common feature
  • Version management and traffic splitting are standard
  • Monitoring and observability are integrated

Platform Differences

  • SageMaker offers more fine-grained control over containers
  • Vertex AI provides tighter integration with Google Cloud services
  • Scaling granularity varies between platforms
  • Cost optimization strategies differ significantly
